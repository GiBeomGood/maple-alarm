Sprint Document

Rust Alarm App

rust · nwg · windows-sys · target: windows x86_64 · single-binary
00

프로젝트 목적

메이플스토리 버프 스킬 재사용 리마인더. 게임 내 2분 지속 버프가 만료되기 전에 재사용해야 하는데, 전투에 집중하다 보면 놓치기 쉬움. 이 도구는 지속시간보다 20초 짧은 100초마다 알람을 울려 재사용 시점을 알림.

확인 버튼 클릭 = 스킬 재사용 시점. 따라서 클릭 응답 지연이 체감 가능한 수준이면 안 됨(알람이 계속 울리는 동안 게임에 집중 불가). pip 창은 게임 위에 띄우되 최소 크기. 단일 .exe, 외부 DLL 없음, 저사양 리소스 점유가 핵심 제약.

⚠ 게임은 창모드 또는 테두리 없는 창모드로 실행할 것. 배타적 전체화면(exclusive fullscreen)에서는 Windows의 TOPMOST 플래그도 덮이므로 pip가 보이지 않음. 이 조건은 사용자가 지키는 전제로 가정하며, 프로그램이 감지·우회하지 않음(비용 대비 가치 낮음).
01

요구사항 정의

  • core프로그램 실행 후 100초 경과 시 알람 발동 (2분 지속 버프 - 20초 여유분. 테스트 시 DEBUG_TIMER 환경변수로 단축)
  • core알람 발동 시 Windows Beep() API로 소리 재생 — 확인 버튼 클릭 전까지 루프
  • core확인 버튼 클릭 시 확인음 재생(알람과 명확히 다른 톤) 후 타이머 100초 리셋 → 반복
  • corepip 창 — 항상 표시, 드래그 가능, topmost, 고정 크기(평상시/알람 동일). 초기 위치는 우측 하단, 드래그로 이동한 위치는 프로세스가 살아있는 동안만 유지(디스크 저장 없음, 재실행 시 다시 우측 하단)
  • core알람 중 확인 버튼은 빨간색으로 깜빡임(0.5초 주기)으로 주의 끌기. 구현은 Frame+Label 합성 버튼(Button 위젯이 아님 — 배경색 커스터마이징 제약 때문)
  • core시스템 트레이 등록 — 첨부 .ico 아이콘 사용, 우클릭 메뉴(남은 시간 + 종료)
  • core다중 인스턴스 방지 — 이미 실행 중이면 기존 창 활성화 후 새 프로세스 종료
  • core시스템 절전(sleep/suspend) 진입 시 프로세스 자동 종료
  • perfRAM 목표 <15MB, CPU 상시 ~0% (릴리즈 빌드 기준)
  • build빌드 타깃: Windows x86_64, cargo build --release, 단일 .exe

03

스레드 아키텍처

main thread — nwg 이벤트 루프. UI, 트레이, 전원 이벤트 수신은 모두 여기서.
├─ timer_thread — 1초 sleep 루프, 카운트다운만 수행
│ Arc<SharedState> 공유 → nwg::Notice로 UI 갱신 신호
├─ beep_thread — Alarm 진입 시 스폰, AtomicBool로 종료 제어
│ Beep()은 동기 블로킹 → duration 120ms로 짧게 잡아 반응성 확보
└─ confirm_beep_thread — 일회성, 확인 클릭 시 더블비프 후 자동 종료
detached — join 불필요
STATE MACHINE
Idle(n초)
─ n=0 ─▶
Alarm
─ 확인 버튼 ─▶
Confirming
─ 확인음 끝 ─▶
Idle(100)

• Idle: timer_thread가 1초마다 remaining 감소 + notice 발신
• Alarm 진입: remaining=0 도달 순간 notice 1회 발신 + beep_thread 스폰 + blink_timer 시작. 이후 UI는 blink_timer가 책임짐(timer_thread는 sleep만)
• Confirming: 확인 클릭 즉시 진입. beep 중단 → 확인음 재생 → 즉시 Idle(reset_secs) 복귀
• 확인음은 비동기 — 끝나기 전에도 이미 Idle 카운트다운 시작됨

04

공유 상태 구조체

// src/state.rs
use std::sync::atomic::{AtomicBool, AtomicU64, Ordering};
use std::sync::Arc;

// 상태는 AtomicU64 하나로 충분 — Mutex 불필요, 데드락 위험 제거
// remaining=0 이면 Alarm, 그 외는 Idle(remaining)로 해석
pub struct SharedState {
    pub remaining_secs: AtomicU64,      // 남은 시간(0 = 알람 중)
    pub alarm_active: AtomicBool,       // beep_thread 루프 제어
    pub reset_secs: u64,                 // 리셋값 (기본 100, DEBUG_TIMER 시 10)
}

impl SharedState {
    pub fn new(reset_secs: u64) -> Arc<Self> {
        Arc::new(Self {
            remaining_secs: AtomicU64::new(reset_secs),
            alarm_active: AtomicBool::new(false),
            reset_secs,
        })
    }
    pub fn is_alarming(&self) -> bool {
        self.remaining_secs.load(Ordering::Acquire) == 0
    }
}
Mutex 제거 이유: 단일 u64 + bool은 원자연산으로 충분. 데드락 리스크·락 순서 고민 자체를 제거.

07

모듈별 구현 명세

main.rs

1. 최상단: #![windows_subsystem = "windows"] (콘솔 창 숨김)
2. instance::acquire_lock() 호출 — 이미 실행 중이면 기존 창을 foreground로 올리고 std::process::exit(0)
3. unsafe { SetProcessDPIAware(); } — 시스템 DPI 대응(인자 없음, 가장 단순한 형태). 반드시 nwg::init() 이전에 호출.
4. let reset = std::env::var("DEBUG_TIMER").ok().and_then(|s| s.parse().ok()).unwrap_or(100);
5. let state = SharedState::new(reset);
6. nwg::init() → ui::AlarmApp::build_ui(...) (nwg-derive)
7. 창 생성 후 초기 위치를 우측 하단으로 배치 — SystemParametersInfoW(SPI_GETWORKAREA)로 작업 영역 조회, 창 크기만큼 빼고 12px 여백
8. timer::spawn(state.clone(), ui_notice_sender)
9. power::register_handlers(hwnd) — pip 창 hwnd에 WM_POWERBROADCAST 서브클래싱
10. nwg::dispatch_thread_events() — 메인 루프 진입

state.rs

섹션 04 코드 그대로. AtomicU64 + AtomicBool만 사용. Ordering::Acquire/Release로 쓰기·읽기 순서 보장. Mutex 없음 → 데드락 불가능.

timer.rs

pub fn spawn(state: Arc<SharedState>, notice: nwg::NoticeSender)
루프 내부:
  thread::sleep(Duration::from_secs(1));
  let cur = state.remaining_secs.load(Acquire);
  if cur == 0 { continue; }  // Alarm 중 — 감소·notice 모두 생략(UI는 blink_timer가 담당)
  let next = cur - 1;
  state.remaining_secs.store(next, Release);
  if next == 0 { state.alarm_active.store(true, Release); alarm::start_beep_loop(state.clone()); }
  notice.notice();  // 메인 스레드에 "UI 갱신하라" 신호

ui.rs

#[derive(NwgUi)]로 AlarmApp 구조체 선언:
• window: nwg::Window — 160×72 논리 픽셀, WS_EX_TOPMOST | WS_EX_TOOLWINDOW, 보더리스(no_min_max, no_system_menu). 타이틀: "AlarmApp_v1_singleton_marker"(single-instance 검출용 고유 제목, 보더리스라 화면에 안 보임)
• label_caption: nwg::Label — "다음 알람까지" 소제목
• label_time: nwg::Label — "M:SS" 큰 시간 표시
• btn_frame: nwg::Frame — 가짜 버튼의 배경(Frame은 background_color 설정 가능)
• btn_label: nwg::Label — btn_frame 위에 겹친 "확인" 텍스트. 클릭 이벤트는 label의 OnLabelClick으로 받음
• tray: nwg::TrayNotification — assets/icon.ico 임베드(include_bytes!)
• tray_menu: nwg::Menu(popup: true) — 우클릭 시 매번 재구축됨
• tick_notice: nwg::Notice — timer_thread가 깨우는 시그널
• blink_timer: nwg::AnimationTimer — 500ms 주기, 알람 중에만 활성

이벤트 매핑:
• OnNotice(tick_notice) → refresh_ui(): label_time 텍스트 갱신, tray tooltip(set_tip)도 같이 갱신
• OnLabelClick(btn_label) → on_confirm(): is_alarming일 때만 동작. alarm_active=false → remaining_secs=reset_secs → blink_timer stop → btn_frame 원색 복귀 → confirm_beep → UI refresh
• OnMousePress(window, MousePressLeftDown) → 드래그 구현 (아래 UI 명세 참조)
• OnContextMenu(tray) → 매번 tray_menu 재구축: 기존 menu item 전부 제거(또는 새 Menu 인스턴스로 교체) → 현재 남은 시간으로 "남은 시간: M:SS" MenuItem(disabled=true) + "종료" MenuItem 생성 후 tray_menu.popup(cursor_x, cursor_y)
• OnMenuItemSelected(exit_item) → nwg::stop_thread_dispatch()
• OnTimerTick(blink_timer) → btn_frame 배경색 토글(#ff4444 ↔ #661111)
• OnWindowClose(window) → 아무것도 하지 않음(닫기 무시) — Alt+F4로는 종료되지 않음. 종료는 트레이 메뉴로만.

alarm.rs

pub fn start_beep_loop(state: Arc<SharedState>):
  thread::spawn(move || { while state.alarm_active.load(Acquire) { unsafe { Beep(1200, 80); } thread::sleep(Duration::from_millis(60)); } });

pub fn play_confirm_sound() — detached 스레드:
  thread::spawn(|| unsafe { Beep(1800, 60); Beep(2400, 60); });

톤 구분: 알람 = 1200Hz 반복(게임 BGM 위로 뚫고 나오는 대역), 확인 = 1800→2400Hz 상승 더블비프.
Beep duration 80ms의 근거: Beep()은 동기 블로킹 → 확인 클릭 후 최대 80ms 지연으로 알람 종료. 게임 조작 리듬을 깨지 않는 수준.

instance.rs

pub fn acquire_lock() -> bool:
  unsafe { CreateMutexW(null, TRUE, w!("Global\\AlarmAppSingletonMutex_v1")) }
  if GetLastError() == ERROR_ALREADY_EXISTS { focus_existing_window(); return false; }
  true

focus_existing_window(): 고유 창 제목으로 기존 창 찾기:
  let hwnd = unsafe { FindWindowW(null(), w!("AlarmApp_v1_singleton_marker")) };
  if !hwnd.is_null() { unsafe { ShowWindow(hwnd, SW_RESTORE); SetForegroundWindow(hwnd); } }

전제: ui.rs에서 pip Window 생성 시 title을 "AlarmApp_v1_singleton_marker"로 고정해야 함. 보더리스 창이라 화면에는 표시되지 않음.
Mutex 핸들은 프로세스 종료 시 OS가 자동 해제 — 명시적 close 불필요.

power.rs

pub fn register_handlers(hwnd: HWND):
nwg window에 raw event handler 바인딩 (nwg::bind_raw_event_handler).
WM_POWERBROADCAST 수신 + wparam == PBT_APMSUSPEND (0x0004) → nwg::stop_thread_dispatch()로 정상 종료 경로 진입.
resume 시에는 어차피 프로세스가 죽어있으므로 처리 불필요.

종료
  • sizepip 창 고정 크기 160×72 논리 픽셀 (DPI 100% 기준). DPI 변경 시 GetDpiForWindow로 재계산
  • pos초기 위치: 우측 하단(작업표시줄 위 12px 여백). SystemParametersInfoW(SPI_GETWORKAREA)로 작업 영역 조회. 드래그로 이동한 좌표는 프로세스 메모리에만 유지되며, 재실행 시 다시 우측 하단
  • drag타이틀바 없이 드래그: window의 OnMousePress(MousePressLeftDown) 핸들러에서 unsafe { ReleaseCapture(); SendMessageW(hwnd, WM_NCLBUTTONDOWN, HTCAPTION as WPARAM, 0); }
  • blink알람 중 500ms 간격 btn_frame.set_background_color([0xff,0x44,0x44]) ↔ [0x66,0x11,0x11] 토글. Idle 복귀 시 즉시 중단 + 원래 비활성 색상([0x22,0x22,0x22]) 복귀
  • zorder일반 창(크롬, 에디터, 탐색기 등)은 non-topmost이므로 pip가 항상 위. 다른 topmost 창과는 경쟁이지만 대부분 pip가 상단 유지. 게임이 창모드·테두리 없는 창모드라면 pip가 게임 위에 보임. 배타적 전체화면은 예외(프로젝트 목적 섹션 참조)
  • dpi첨부 .ico가 30×26 단일 사이즈 — 고DPI 트레이에서 흐릿함. 16×16, 20×20, 24×24, 32×32, 40×40, 48×48 멀티 사이즈 .ico로 재생성 권장 (ImageMagick: magick icon.png -define icon:auto-resize=48,40,32,24,20,16 icon.ico)

구현 주의사항

  • criticalnwg UI 컴포넌트는 반드시 메인 스레드에서만 생성·접근. 워커 스레드 → UI 갱신은 nwg::Notice 통해서만.
  • criticalbeep_thread는 alarm_active만 검사 — Beep duration 80ms 내 종료 보장. JoinHandle 보관 불필요(자연 종료).
  • critical확인 클릭 핸들러 순서: alarm_active.store(false) → remaining_secs.store(reset_secs) → blink_timer stop → btn_frame 원색 복귀 → confirm_beep 재생 → UI refresh. 순서 역전 시 beep이 늦게 멈추거나 UI가 한 프레임 늦게 갱신됨.
  • dpiSetProcessDPIAware()는 nwg::init() 이전에 호출해야 효과 있음. 인자 없는 함수라 가장 단순.
  • panic릴리즈 프로파일에 panic = "abort" 설정. 어떤 스레드든 패닉하면 프로세스 통째로 종료 — 좀비 백그라운드 프로세스 방지.
  • criticalnwg MenuItem에는 런타임 set_label / set_text setter가 존재하지 않음. "남은 시간" 표시는 OnContextMenu 핸들러에서 매번 메뉴를 새로 빌드해 popup. 재빌드 비용은 우클릭 시점 1회라 무시 가능.
  • closepip 창 X 버튼 없음(보더리스). Alt+F4는 OnWindowClose 핸들러에서 무시 처리 → 종료되지 않음. 종료는 오직 트레이 메뉴 "종료"를 통해서만.
  • game게임이 배타적 전체화면이면 WS_EX_TOPMOST도 덮여서 pip 안 보임. 창모드/테두리 없는 창모드에서만 동작. 이 프로그램은 감지·우회 시도하지 않음 — 사용자가 설정으로 해결.
  • envDEBUG_TIMER는 프로세스 시작 시점 1회만 읽음. 런타임 변경 불가(단순화).
